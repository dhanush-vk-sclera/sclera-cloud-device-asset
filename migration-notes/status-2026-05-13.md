## Key gotchas discovered

1. **PowerShell `Set-Content -Encoding utf8` writes UTF-8 BOM** that breaks javac on `.java` files (bit T7; fixed in `e96e163`). Memory: `feedback_powershell-utf8-bom.md`.
2. **JDK 17 + Lombok 1.18.22** (Spring Boot 2.6.5 default) does not generate `@Data`/`@Builder` methods reliably in `mvn compile`, even though direct `javac` with explicit `-processorpath` does. Workarounds tried: upgrade to 1.18.30/1.18.36, add `<annotationProcessorPaths>`, `<fork>true</fork>`, `--add-opens` flags, `-source`/`-target` vs `<release>` — all unsuccessful in isolation. Workaround that worked: **delombok the tree**, converting @Data/@Builder to explicit methods.
3. **`mvnw.cmd` requires `JAVA_HOME` set**. Not set globally on this machine; must export per shell.
4. **Maven not on PATH**. Located at `C:\Users\DhanushVasanth\.tools\apache-maven-3.9.15\bin\`.
5. **JDK 11 not installed**. Only JDK 17 and JDK 21 available — but `<target>11</target>` works fine.
6. **`@DataJpaTest` H2 fallback**: the eventual repo tests may need Testcontainers if `DeviceRepository`'s native MySQL SQL doesn't parse in H2-MySQL-compat mode (deferred to T21).
7. **`application-test.yml` JWT issuer-uri unreachable** (`http://localhost/test-issuer`) — will break T18 context-load smoke test unless a `TestSecurityConfig` stubs `JwtDecoder` in test scope.
